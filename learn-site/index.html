From Zero to Full-Stack & AI Developer

A friendly, step-by-step plan for a complete beginner — with a video for every single step.

Hi! Read this first. 👋

This is your map from knowing nothing to building real websites and AI. Don’t rush. Do the steps in order — each one needs the one before it. Watch the Main video in each phase first, type the code yourself, then build the little project. That is the whole secret.

You will feel confused sometimes. Everyone does — even experts. Confused just means your brain is growing. Keep going. 💪

⏳ How long does this take? Be honest with yourself.

This is a ~1.5 to 3 year journey, depending on how much time you give it. That is normal — real skills take real time, no matter what the ads promise. But you don’t wait years to feel proud: you’ll put your first website online in about 6 months, and build your first complete app in about a year. The rest is you getting stronger. Consistency beats speed — one hour most days will carry you further than long weekends once a month.

💸 Everything on this page is free. You do not need to buy a single course. The few platforms worth knowing — freeCodeCamp (free certificates), LeetCode (practice), and Kaggle (free AI datasets) — are all free too. If anyone tells you that you must pay thousands for a “coding academy” to learn this, they are selling something. Your effort is the only price.

Three words you’ll see a lot

  • Frontend — the part of the app you see and click, in the browser.
  • Backend — the hidden part on a server that does the thinking and stores data.
  • Full-stack — being able to build BOTH the frontend and the backend.

Good news: one project that grows

You don’t build a brand-new big thing every phase. You build the same little To-Do app again and again — each time a bit bigger. Much less scary.

How to use every video

Watching is not the same as learning. Use this loop every time: 1) Watch a small part → 2) Type the code yourself → 3) Rebuild it from memory with the video closed → 4) Stuck? Re-watch that bit (totally fine), then go round again.

🇵🇰 If English is hard at first: CodeWithHarry and Chai aur Code (Hitesh Choudhary) teach the same things in Hindi/Urdu. Use them as extra help — and keep improving your English too, because it opens many doors in tech.

🤖 Use AI as a tutor — not as a crutch.

Free AI tools (ChatGPT, Claude, Gemini) are like having a patient teacher beside you 24/7. Used well, they make you learn faster. Used lazily, they make sure you learn nothing. The difference is simple:

  • ✅ Paste your error message and ask: “explain what this means in simple words.” Then fix it yourself.
  • ✅ Ask it to quiz you: “ask me 5 questions about Python loops.”
  • ✅ Ask why: “why is my code slow?”, “what is a better way to write this?”
  • ❌ Don’t say “write my whole project.” You’ll feel fast and learn nothing — and you won’t be able to fix it when it breaks.

The rule: understand it first, then use AI to go faster. AI replaces people who never understood. It makes people who do understand unstoppable. (There’s also a small AI helper on this page — see the button at the bottom right.)

The phases

Phase 0 Get Ready 1–2 weeks

Before you can run, you set up your shoes. This phase just gets your computer ready so nothing annoys you later.

📚 What to learn

  • Install Python and VS Code (= the program where you write your code).
  • Learn the terminal (= a text window where you type commands to tell the computer what to do).
  • Practise typing without looking at the keyboard (use monkeytype.com).
  • Learn to copy an error message into Google and read the answers. This is a real skill — even experts do it every day.
  • Most important: feeling confused is normal. It does not mean you are not clever. It means you are learning.
💻 No powerful computer? You can still start today. A slow or old laptop is totally fine for everything up to Phase 8. Only have a phone, or no laptop at all? Write and run real code right in your browser for free with Replit (replit.com) for Python and web, and Google Colab (colab.research.google.com) for Python and the AI maths later — no install needed. A weak machine is never a reason to quit.

▶ Main — Harvard CS50, Lecture 0 (what is programming?) ▶ freeCodeCamp — Terminal / command line for beginners

✍️ Practice (small things to build)

  • Install Python and make it print the words Hello, world.
  • In the terminal: make a folder, put a file in it, show what is inside, then delete it.
  • Do 3 short typing lessons.
  • Type one wrong letter on purpose, read the red error, and Google it.

🛠️ Project

A tiny Python program that asks your name and favourite subject, then prints a pretty "hello card". The goal is just to see your setup working.

✅ Ready for the next phase when: You can open VS Code, type a small program, and run it in the terminal without help.
Phase 1 Python — the foundation 2–4 months

Python is your first language. It reads almost like English, so it is the kindest place to start. It is also the language the whole world uses for AI.

📚 What to learn

  • Basic syntax (= the spelling and grammar rules of a language): variables, numbers, text, input and output.
  • Making choices and repeating things: if / else, for, while.
  • Ways to hold many things: lists, dictionaries, sets.
  • Functions: a block of code you can reuse, so you don’t write the same thing twice.
  • Files: reading and writing text files.
  • Mistakes: how to catch errors (try / except) and read what they say.
  • OOP (= a way to group code into "objects" that hold both data and actions). This one feels hard at first — that is completely normal, go slow.

▶️ Watch (click to open)

▶ Main — Programming with Mosh: Python for Beginners (≈6 hrs) ▶ freeCodeCamp — Learn Python, Full Course ▶ Corey Schafer — Python Tutorials (for harder topics like OOP) ▶ 🇵🇰 Hindi/Urdu — CodeWithHarry: The Ultimate Python Course

✍️ Practice (small things to build)

  • FizzBuzz (print numbers, but with a twist).
  • A program that turns Celsius into Fahrenheit.
  • Count the vowels in a word.
  • A small phone-book using a dictionary.
  • Read a text file and count how many words it has.

🛠️ Project

A To-Do app in the terminal: add tasks, see them, mark them done, delete them, and save them to a file so they are still there next time. Remember this app — you will rebuild it bigger later!

✅ Ready for the next phase when: You can write a small program from a blank file by yourself, without copying a video line-by-line.
Phase 2 Computer science basics 3–6 months

These are the thinking skills behind coding. They are exactly the part that AI cannot do for you — so they make you valuable.

📚 What to learn

  • Git (= a tool that saves every version of your code so you can undo mistakes) and GitHub (= a website that keeps your code online). Every project from now lives in a repo (= a project folder Git watches).
  • Ways to store data: lists, stacks, queues, hash maps (= super-fast "name → value" lookups), and later trees and graphs.
  • Step-by-step methods (algorithms): searching, sorting, and recursion (= a function that calls itself to solve a smaller piece of the same problem).
  • Big-O (= a simple way to say how much slower your code gets when the data gets big).

▶️ Watch (click to open)

▶ Main — Harvard CS50 (full course, free) ▶ freeCodeCamp — Data Structures and Algorithms ▶ Traversy Media — Git & GitHub crash course ▶ NeetCode — how to solve coding problems

✍️ Practice (small things to build)

  • Write a "binary search" (a fast way to find a number in a sorted list).
  • Reverse a word without using the built-in reverse.
  • Solve 10 Easy problems on LeetCode (Easy only — be kind to yourself).
  • Put all your old projects on GitHub, each with a short README (= a note that says what the project does).

🛠️ Project

A GitHub page with 15–20 small problems you solved, sorted by topic. This is the start of your "proof" that you can code.

✅ Ready for the next phase when: You can save and upload code to GitHub on your own, and you understand why one solution is faster than another.
Phase 3 Maths for AI slow, in the background

This is the secret weapon. Most people skip it — and that is exactly why most people can use AI but cannot build it. Go slowly, a little at a time, next to your school maths.

📚 What to learn

  • Linear algebra (= the maths of vectors and matrices). A vector (= a list of numbers); a matrix (= a grid of numbers). This is the language AI speaks.
  • Calculus — slopes and gradients (this is literally how AI "learns").
  • Probability and statistics — averages, spread, and chance.
💡 You can be a great website developer with almost no advanced maths. But to truly BUILD AI (not just use it), this maths is a must. No rush — just keep nibbling at it.

▶️ Watch (click to open)

▶ Main — 3Blue1Brown: Essence of Linear Algebra (beautiful, makes it click) ▶ 3Blue1Brown — Essence of Calculus ▶ StatQuest — Statistics, made simple ▶ Khan Academy — for step-by-step practice

✍️ Practice (small things to build)

  • By hand, add and multiply two small (2×2) grids of numbers.
  • Find the slope (derivative) of x² + 3x.
  • Find the average and spread of a few numbers.
  • Do the same grid maths in Python with NumPy (= a Python tool for fast number maths).

🛠️ Project

A short Python notebook that opens a small data file, finds the averages, and draws a simple chart with Matplotlib (= a Python tool for making charts).

✅ Ready for the next phase when: You are not scared of a matrix or a graph, and you can do small maths both by hand and in Python.
Phase 4 Web basics: HTML, CSS, JavaScript 3–4 months

Now you make things people can actually SEE in a web browser. We use the plain building blocks first — no shortcuts yet, so you really understand how the web works.

📚 What to learn

  • HTML (= gives a web page its bones — the headings, text, buttons, forms).
  • CSS (= the paint and layout — colours, spacing, and making it look good on a phone too). This "looks good on any screen" idea is called responsive design.
  • JavaScript (= makes the page DO things when you click or type). You’ll meet the DOM (= the live page that JavaScript can change) and async code (= code that waits for slow things, like the internet, without freezing the page).

▶️ Watch (click to open)

▶ Main — freeCodeCamp: Responsive Web Design (HTML & CSS) ▶ Kevin Powell — the CSS master (for anything that confuses you) ▶ Web Dev Simplified — JavaScript made simple ▶ The Net Ninja — Modern JavaScript (playlist) ▶ Traversy Media — 50 Projects in 50 Days (lots of practice)

✍️ Practice (small things to build)

  • Build a simple recipe page.
  • Make 3 cards sit side by side (try Flexbox, then Grid).
  • Add a button that switches between dark and light colours.
  • Pull a joke from a free API (= a way for two programs to share data) and show it on the page.

🛠️ Project

Your own portfolio website (about you), with one fun interactive thing, put online for free with GitHub Pages. To deploy (= put your work on the internet for anyone to see) — that is a big, exciting moment. Your first thing on the real web!

✅ Ready for the next phase when: You can build a small web page from scratch and put it online with a real link you can share.
Phase 5 React — a frontend framework 2–4 months

This is how real companies build modern websites. Start ONLY after plain JavaScript feels okay — React is much easier when the basics are solid.

📚 What to learn

  • React (= a popular JavaScript framework — a ready-made toolbox — for building screens out of small reusable pieces).
  • Components (= the small reusable pieces), props (= info you hand to a piece), and state (= info a piece remembers and can change).
  • Hooks like useState and useEffect (= special React helpers that give pieces powers like memory).
  • Tailwind CSS (= a fast way to style with tiny labels) and later TypeScript (= JavaScript that warns you about mistakes early).

▶️ Watch (click to open)

▶ Main — The Net Ninja: Full Modern React Tutorial (React 19) ▶ Traversy Media — React crash course ▶ JavaScript Mastery — build real React projects ▶ Traversy Media — TypeScript crash course (later)

✍️ Practice (small things to build)

  • A counter with + and − buttons.
  • A list where you can add and delete items.
  • Show data from an API, with a "loading…" message and an error message.
  • A small 3-page site you can click between.

🛠️ Project

A weather app or movie-search app in React that uses a real API, has pages, and is styled with Tailwind. Put it online with Vercel (= free hosting for the front of a website).

✅ Ready for the next phase when: You can build a multi-page React app that fetches real data and shows it nicely.
Phase 6 Backend — the server side 3–4 months

The backend is the hidden brain: it does the thinking and remembers the data. We stay in Python, so you build on what you already know.

📚 What to learn

  • What a backend does: it gets a request, thinks, talks to the database, and sends an answer back.
  • REST API (= a simple, standard way for programs to ask each other for things) using the actions GET, POST, PUT, DELETE.
  • CRUD (= the four basic things you do to data: Create, Read, Update, Delete).
  • Login, using a token / JWT (= a secure digital pass that proves who is logged in).
  • Pick one tool: FastAPI (= modern, fast, and great for AI — recommended) or Django (= bigger, comes with lots built in).

▶️ Watch (click to open)

▶ Main — freeCodeCamp: FastAPI / Python API Development ▶ freeCodeCamp — Django full course ▶ The Net Ninja — Django (playlist) ▶ Official FastAPI tutorial (one of the best anywhere)

✍️ Practice (small things to build)

  • A tiny API that answers "hello".
  • Add Create / Read / Update / Delete for a list of items.
  • Add a simple login with a token.
  • Open your API’s automatic test page (called Swagger) and try it.

🛠️ Project

Turn your Phase 1 To-Do app into a real online API with full CRUD and a login. Same app — now living on a server!

✅ Ready for the next phase when: You can build an API that stores items and handles a basic login.
Phase 7 Databases 1–2 months

Every real app needs to remember things even after it is closed. That is what a database is for.

📚 What to learn

  • SQL (= the language for asking a database questions): SELECT, WHERE, INSERT, UPDATE, DELETE, and JOIN (= joining two tables together).
  • PostgreSQL (= a popular, powerful, free database).
  • Good table design and normalization (= not storing the same thing twice).
  • ORM (= a tool that lets you use the database with normal code instead of database commands).
  • Easy modern option: Supabase (= a ready-made online database with login built in).

▶️ Watch (click to open)

▶ Main — freeCodeCamp: SQL Full Course ▶ The Net Ninja — SQL & MongoDB (playlists) ▶ Supabase — crash course

✍️ Practice (small things to build)

  • On paper, design 3 tables (users, posts, comments) and draw how they connect.
  • Write a question that finds and sorts some rows.
  • Write a JOIN across two tables.
  • Connect your Phase 6 API to a real database.

🛠️ Project

Add a real database to your backend so the data stays after restart — and link it (one user has many tasks).

✅ Ready for the next phase when: Your To-Do app keeps its data forever, in a real database, linked to each user.
Phase 8 Full app, online ⭐ 2–3 months

The big one. You join the front, the back, and the database into ONE real app and put it on the internet. After this, you are a full-stack developer.

📚 What to learn

  • Connect the three parts: React (front) ↔ your API (back) ↔ database.
  • Build one complete app with login, from start to finish.
  • Put it online: front on Vercel, back on Render or Railway (= free hosting for backends), database online.
  • Keep secrets in environment variables (= passwords kept outside your code).

▶️ Watch (click to open)

▶ Main — JavaScript Mastery: full-stack project (front to back, deployed) ▶ freeCodeCamp — full-stack course ▶ Fireship — short, clear deployment videos

✍️ Practice (small things to build)

  • Connect your React app to your own API.
  • Hide pages from people who are not logged in.
  • Move all secrets into environment variables.
  • Do one real deploy — front AND back.

🛠️ Project

⭐ YOUR BIG PROJECT: one complete, online app that solves a small real problem (a study-notes app, an expense tracker, a habit tracker). It must have a login, a database, a React front, an API back, a live link anyone can open, and a clear README with screenshots. This one project is worth more than ten tutorials.

✅ Ready for the next phase when: You have ONE finished app online that you can send to anyone as a link. You are now full-stack capable.
Phase 9 The AI layer ongoing

This is the future you want. Do it in two parts — the fun, useful part first, then the deep part.

📚 What to learn

  • Part A — using AI (start around Phase 5–6): learn prompt engineering (= writing good instructions so the AI gives good answers), and add an AI feature (like a chatbot) to your Phase 8 app using an LLM (= a big AI model that reads and writes text, like ChatGPT or Claude).
  • Part B — understanding AI (the long game, needs the Phase 3 maths): learn Pandas (= a Python tool for tables of data), then machine learning with scikit-learn, then neural networks (= a brain-inspired system that learns from examples) with PyTorch.
💡 Order matters: enjoy USING AI in apps first. Go deep on BUILDING AI later, once the maths is ready. Doing the hard maths first is how most people give up.

▶️ Watch (click to open)

▶ Main (Part B) — Andrej Karpathy: Neural Networks, Zero to Hero ▶ Part A — DeepLearning.AI free short courses (Prompt Engineering, RAG) ▶ 3Blue1Brown — Neural Networks (the intuition) ▶ StatQuest — Machine Learning ▶ Andrew Ng — Machine Learning (the classic)

✍️ Practice (small things to build)

  • Write 5 different prompts for one task and compare the answers.
  • Train a tiny model on a beginner dataset (Iris or Titanic) and chart it.
  • Follow Karpathy’s first video to build a tiny neural network.

🛠️ Project

(A) Add a real AI feature to your Phase 8 app. (B) Later, train a model on a free Kaggle (= a website with free datasets) dataset and write up what you found.

✅ Ready for the next phase when: You have shipped one real AI feature in an app, and you understand the basic idea of how a model learns.
Phase 10 Keep growing forever

You never really finish — and that is the fun part. Keep building, keep sharing, keep learning.

📚 What to learn

  • Testing, clean code, and a little system design (how big apps are built).
  • Open source (= public projects anyone can help improve) — make your first small contribution on GitHub.
  • A strong portfolio: a tidy GitHub plus a personal site with 3–4 real projects.
  • Keep improving your English — most of the best material is in English, and it opens doors.

▶️ Watch (click to open)

▶ Fireship — what’s new in tech ▶ ThePrimeagen — a senior developer’s view ▶ NeetCode — interviews & system design ▶ ArjanCodes — clean Python

✍️ Practice (small things to build)

  • Write one blog post or README explaining something you built.
  • Make one small open-source contribution.
  • Keep your GitHub green — code a little, often.

🛠️ Project

Keep your portfolio fresh: 3–4 real projects, each with a live link and a clear README.

✅ Ready for the next phase when: You can learn any new tool on your own now — that is the real skill.

🙂 A few honest words

  • You’ll often feel “not smart enough.” That feeling is normal and it is not the truth. Keep going.
  • Don’t chase every shiny new tool. Master the basics — everything else gets easy after that.
  • AI will tempt you to skip the hard understanding. Don’t. Use AI to go faster AFTER you understand. AI replaces people who never understood; it makes people who DO understand much stronger.
  • Some weeks feel great, some feel stuck. Just keep showing up. That is how everyone makes it.

🚪 Where this can take you

This isn’t just a hobby — these are the exact skills companies pay for. Once you’ve finished even Phase 8 (one full app online), real doors open:

  • Freelancing — build small websites and apps for local businesses, or on Upwork and Fiverr, from anywhere with internet.
  • A junior developer job — your portfolio of real, deployed projects is your proof; it matters more than any certificate or even a degree.
  • Open source — help improve real projects on GitHub, meet developers worldwide, and build a name for yourself.
  • Building your own thing — the same skills let you turn your own idea into a real product one day.

You don’t need to live in a big city or know the “right” people. You need a working internet connection, this path, and the patience to keep showing up.

🎯 The whole plan in one line

Python → think like a programmer → CS basics → web (HTML/CSS/JS) → React → backend → databases → build one full app online → then go deep on AI.
Do a little maths in the background, and finish one small project at the end of every phase.

You have time, a clear goal, and a path many people have walked before you. The only thing that matters now is starting Phase 0 — and not stopping. You’ve got this. 🚀
